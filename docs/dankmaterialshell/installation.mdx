## Distribution Packages

### Arch Linux

#### Stable release
```bash
paru -S dms-shell-bin
```
#### Latest development build
```bash
paru -S dms-shell-git
```

These packages ship the shell, widgets, and CLI. Pair them with the `niri` or `hyprland` packages from the official repositories for a complete desktop stack.

### Fedora / RHEL / Rocky / Alma

Enable the COPR repositories managed by the team to install prebuilt packages:


#### Stable release
```bash
sudo dnf copr enable avengemedia/dms
sudo dnf install dms
```

### Latest development build
```bash
sudo dnf copr enable avengemedia/dms-git
sudo dnf install dms
```

The COPR repositories also provide companion packages such as `quickshell-git`, `cliphist`, `ghostty`, and other utilities used by the default configuration.

### NixOS (via flakes)

NixOS has its own documentation, see [here](./nixos.mdx) for details.

## Manual Installation (All Distros)

Use the manual path when packaging for a new distribution or running a stripped-down environment:

1. **Install Quickshell**
   ```bash
   # Arch
   paru -S quickshell-git

   # Fedora
   sudo dnf copr enable avengemedia/danklinux
   sudo dnf install quickshell-git
   ```
   Build from source on Debian-based systems where packages are unavailable.

2. **Clone the shell sources**
   ```bash
   mkdir -p ~/.config/quickshell
   git clone https://github.com/AvengeMedia/DankMaterialShell.git ~/.config/quickshell/dms
   ```

3. **Install the CLI manager**
   ```bash
   sudo sh -c "curl -L https://github.com/AvengeMedia/danklinux/releases/latest/download/dms-$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/').gz | gunzip | tee /usr/local/bin/dms > /dev/null && chmod +x /usr/local/bin/dms"
   ```

4. **Install optional integrations** for full functionality (see below).

After completing the manual steps, launch the shell with `dms run`. Use `dms restart` or `dms kill` to manage the session during upgrades.

## Optional Components

While DankMaterialShell runs without extras, the following tools unlock signature features:

- `dgop`: Enables detailed system metrics, process lists, and temperature widgets.
- `matugen`: Generates Material Design color palettes from your wallpapers.
- `brightnessctl`: Controls display backlight and keyboard lighting.
- `wl-clipboard` + `cliphist`: Drives clipboard history and copy helpers.
- `cava`: Powers the audio visualizer widget.
- `qt6-multimedia`: Provides system sound feedback.

Install them through your package manager or the DankLinux installer. Consult the [Plugins](/docs/dankmaterialshell/plugins) and [Widgets](#) pages to learn how these dependencies are used.

## Post Install Checklist

- Pick your compositor and finish keybinding setup via the [Compositor Setup](/docs/dankmaterialshell/compositors) guide.
- Review [Configuration](/docs/dankmaterialshell/configuration) to tailor widgets, theming, and system preferences.
- Explore optional modules like the [Greeter](/docs/dankmaterialshell/greeter) and the [Plugin Registry](https://plugins.danklinux.com/) for additional functionality.
