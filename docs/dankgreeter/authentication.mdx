# DankGreeter authentication

DankGreeter authenticates logins through greetd's PAM service. DMS can manage
the fingerprint and security-key entries in `/etc/pam.d/greetd`, or you can
make your distribution or system configuration the policy owner.

Configure this in **Settings → Greeter → Authentication**.

## Choose who manages greetd PAM

Leave **greetd PAM is externally managed** off for the normal DMS-managed
setup. The fingerprint and security-key toggles update the shared greetd PAM
configuration automatically.

Turn it on when greetd PAM is managed by `authselect`, `pam-auth-update`,
NixOS, or your own PAM files. DMS then removes its managed block and stops
writing `/etc/pam.d/greetd`; the factor toggles are disabled because the
system PAM configuration decides what the greeter offers.

:::caution

Keep a root terminal or another login method available while changing PAM.
An invalid greetd stack can prevent graphical login.

:::

## Fingerprint login

1. Install your distribution's `fprintd` service and PAM integration.
2. Enroll at least one finger with `fprintd-enroll`.
3. Confirm it with `fprintd-list "$USER"`.
4. Enable **Fingerprint at login** in the Greeter Authentication card.

For example, Fedora uses:

```bash
sudo dnf install fprintd fprintd-pam
fprintd-enroll
```

If enrollment reports **No devices available**, check the
[libfprint supported devices list](https://fprint.freedesktop.org/supported-devices.html).
Some Validity/Synaptics readers instead need
[open-fprintd](https://github.com/uunicorn/open-fprintd) and
[python-validity](https://github.com/uunicorn/python-validity). Do not run the
stock and alternate `fprintd` implementations together.

## Security-key login

Install `pam_u2f`, register the key in the mapping used by your distribution,
and test that mapping before enabling **Security key at login**. Enabling the
toggle does not install `pam_u2f` or register a key.

The lock screen's **OR** and **AND** modes do not change greeter policy. The
greeter follows the ordering and control flags in its own greetd PAM stack.

## Repair and troubleshooting

Normal setting changes apply automatically. Use the following only to
manually repair or reapply DMS-managed configuration:

```bash
dms auth sync
```

The command may open a terminal for sudo and rewrites only the DMS-managed PAM
block.

See [DankGreeter configuration](./configuration) for the rest of the greeter
setup, and [Lock Screen authentication](../dankmaterialshell/lock-screen-authentication)
for the separate in-session lock policy.
