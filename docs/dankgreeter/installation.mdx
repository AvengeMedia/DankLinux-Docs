# Installation

## Installation Methods

:::note[How installation works]

Install the `dms-greeter` package for your distro (sections below), then run:

```bash
dms-greeter install
```

This installs greetd if needed and handles all configuration automatically.

> ⚠️ **[This replaces your current display manager.](#dankinstall-users)**

The old `dms greeter <command>` form still works as a deprecated alias that forwards to `dms-greeter`.

:::

## Distro Installation

### Arch & Derivatives

DankGreeter is available in the AUR as a prebuilt binary package or a source (`-git`) package.

<Tabs groupId="greeter-channel">
<TabItem value="bin" label="Prebuilt Binary" default>

```bash
paru -S greetd-dms-greeter-bin
```

`greetd-dms-greeter-bin` installs a prebuilt binary from the latest tagged release, so there is nothing to compile. This is the fastest way to install and the recommended option for most users.

</TabItem>
<TabItem value="git" label="Latest Development Build">

```bash
paru -S greetd-dms-greeter-git
```

`greetd-dms-greeter-git` builds from the tip of the `master` branch. Use it if you want the newest changes before they are tagged, at the cost of a local compile and possible instability.

</TabItem>
</Tabs>

Both packages ship the `dms-greeter` binary along with the `/usr/lib/sysusers.d/dms-greeter.conf` and `/usr/lib/tmpfiles.d/dms-greeter.conf` declarations described in [Greeter system user](#greeter-system-user-rpm-deb-and-immutable-systems). After installing, either run `dms-greeter install` (see the note at the top) or continue with [Completing Setup](#completing-setup).

DankGreeter is published to the [avengemedia/danklinux](https://copr.fedorainfracloud.org/coprs/avengemedia/danklinux/) COPR from the [dank-greeter](https://github.com/AvengeMedia/dank-greeter) repository.

```bash

```

Tagged releases. Recommended for most users.

</TabItem>
<TabItem value="git" label="Git (development)">

```bash
sudo dnf copr enable avengemedia/danklinux
sudo dnf install dms-greeter-git
```

Builds from the tip of `master` in dank-greeter. Conflicts with the stable `dms-greeter` package (both provide `/usr/bin/dms-greeter`).

## Greeter system user (RPM, DEB, and immutable systems)

Official `dms-greeter` / `dms-greeter-git` packages (Arch AUR, Fedora, Debian, Ubuntu, openSUSE) ship two systemd declarations under `/usr`:

| File | Purpose |
|------|---------|
| `/usr/lib/sysusers.d/dms-greeter.conf` | Declares the `greeter` UNIX account (home `/var/lib/greeter`, shell `/bin/bash`) |
| `/usr/lib/tmpfiles.d/dms-greeter.conf` | Creates `/var/cache/dms-greeter` and `/var/lib/greeter` with `greeter:greeter` ownership |

`systemd-sysusers` applies the user definition at boot and on package install. Package scriptlets (`useradd` / `adduser`) remain as a fallback on traditional mutable systems.

### Immutable and image-based systems

On Fedora Atomic, Universal Blue, bootc, [Zirconium](https://github.com/zirconium-dev/zirconium/), and similar systems:

- Install **`dms-greeter` or `dms-greeter-git` from your distro repo** (layered package or image build). Do **not** rely on `dms-greeter install`, `dms-greeter enable`, or `dms-greeter uninstall` - those commands are disabled on immutable systems by CLI policy.
- After installing or upgrading the greeter package, **reboot once** if `getent passwd greeter` is empty (sysusers runs early in boot).
- Configure `/etc/greetd/config.toml`, enable `greetd`, and set up theme sync using your distro or image documentation. `dms-greeter sync` may still work for theme files if not blocked by your image policy.

Verify the account:

```bash
getent passwd greeter
getent group greeter
ls -l /usr/lib/sysusers.d/dms-greeter.conf
systemd-sysusers --dry-run | grep greeter
```

If you previously added a manual `/usr/lib/sysusers.d/dms-greeter.conf` workaround (for example on Zirconium before upstream packaging), you can remove it once `dms-greeter` from DankLinux includes the same file.

### Gentoo

There is no Gentoo package yet. Install the prebuilt binary from the latest release:

```bash
curl -fsSL https://github.com/AvengeMedia/dank-greeter/releases/latest/download/dms-greeter-linux-amd64.tar.gz | tar -xz
sudo install -Dm755 dms-greeter-linux-amd64 /usr/local/bin/dms-greeter
```

Then run:

```bash
dms-greeter install
```

This will automatically install `greetd` via Portage, configure permissions, and set up greetd to use dms-greeter. To start the greeter immediately without rebooting, run `sudo systemctl start greetd`.

### Void Linux

After installing **`dms-greeter`** or **`dms-greeter-git`** (see [Void Linux](#void-linux) under Distro Installation above), enable and optionally sync theming:

```bash
dms-greeter enable
dms-greeter sync     # optional: share theming with the shell
```

On Void, `dms-greeter enable` sets up everything logind provides automatically on systemd distros: it enables the `dbus` and `elogind` services, configures greetd for elogind (`LIBSEAT_BACKEND=logind`), rewrites `/etc/sv/greetd/run` to wait for those services at boot, adds the `_greeter` user to the `video`/`input` groups, and adds `pam_rundir` to `/etc/pam.d/greetd` so the post-login session gets an `XDG_RUNTIME_DIR`. If `seatd` is enabled it is disabled, elogind manages the seat instead.

On runit, enabling greetd starts it within a few seconds, no separate start command is needed. greetd package updates restore the stock run script, re-run `dms-greeter enable` afterwards.

A Wayland compositor (`niri`, `hyprland`, `sway`, …) is required and is not pulled in automatically.

## Installer & Manual Installation

### DankInstall Users
(Niri and Hyprland users only)

`dms-greeter` is an **opt-in** component in the DankInstaller. It can be toggled on during the installation process, but is skipped by default to ensure users consciously choose to enable it (since it replaces your current display manager).


If you enable `dms-greeter` in DankInstall, the full greeter setup (permissions, ACLs, greetd configuration, service enablement) runs automatically at the end of the install process. No further steps are required.

If you skipped it during dankinstall and want to add it later, install the package for your distro (see the sections above) and then proceed to [Completing Setup](#completing-setup).

:::warning

Enabling `dms-greeter` will replace your current display manager.

:::

### Manual installation (no package)

Prefer your distro package when available (see the sections above). If your distro has no `dms-greeter` package, install `greetd` and `quickshell`, then grab the prebuilt binary from the latest release:

```bash
curl -fsSL https://github.com/AvengeMedia/dank-greeter/releases/latest/download/dms-greeter-linux-amd64.tar.gz | tar -xz
sudo install -Dm755 dms-greeter-linux-amd64 /usr/local/bin/dms-greeter
```

Or build it from source (requires go and make):

```bash
git clone --recurse-submodules https://github.com/AvengeMedia/dank-greeter.git
cd dank-greeter
make build
sudo make install
```

The UI is embedded in the binary, there is no quickshell config tree to install. Create the greeter user and cache directory if your setup does not already have them (see the [dank-greeter README](https://github.com/AvengeMedia/dank-greeter)):

```bash
sudo mkdir -p /var/cache/dms-greeter
sudo chown greeter:greeter /var/cache/dms-greeter
```

Then proceed to [Completing Setup](#completing-setup) below.

:::note

Some distributions may have different user/group names for the greetd user.

:::

#### Prerequisites for Theme Syncing

ACLs (Access Control Lists) are required to allow the greeter user to traverse your home directory and access configuration files. The `acl` package is installed automatically when you run `dms-greeter sync` or `dms-greeter install`, so no manual action is needed on supported distributions.

## Completing Setup

### If you used `dms-greeter install`

This applies to any distro where you ran `dms-greeter install`, including **Arch Linux** (requires paru or yay), **Ubuntu**, **Fedora**, **Debian**, **openSUSE**, and **Gentoo**.

Configuration, permissions, theme syncing, and greetd service enablement are all handled automatically. To start the greeter immediately without rebooting:

```bash
sudo systemctl start greetd
```

You can verify your setup at any time with `dms-greeter status`. `dms-greeter sync` can be re-run after theme changes.

### If you installed the package manually

This applies to: **Arch Linux**, and any other distro where you installed the package manually using the commands above.

After installing the package, run these two commands to complete setup:

#### 1. Enable the Greeter

```bash
dms-greeter enable
```

This will:
- Configure `/etc/greetd/config.toml` with the correct compositor command
- Disable conflicting display managers (gdm, lightdm, sddm)
- Enable and start the greetd service

#### 2. Sync with Your User Theme

```bash
dms-greeter sync
```

This will:
- Install `acl` (if not already installed)
- Add your user to the `greeter` group (if needed)
- Set up ACL permissions on parent directories for greeter access
- Configure group permissions on DMS config directories
- Create symlinks to sync settings, wallpapers, and color themes
- Create your per-user cache slot at `/var/cache/dms-greeter/users/<username>/`

:::note

After running `dms-greeter sync`, you will need to log out and log back in for group membership changes to take effect.

:::

:::tip[Multi-user systems]

One **main admin** runs `dms-greeter sync` once for system setup. Add every other user to the `greeter` group in **Settings → Users** (greeter access toggle), or with `sudo usermod -aG greeter <username>`. Each of those accounts should then run **`dms-greeter sync --profile`** after logging out and back in - not full sync. See [Configuration → Multi-user systems](./configuration#multi-user-systems).

:::

:::note

**NixOS users:** The `dms-greeter enable` and `dms-greeter sync` commands are not available on NixOS. Please follow the manual steps below or see the [NixOS installation guides](./nixos).

:::

### For all users

You can check your greeter configuration at any time:

```bash
dms-greeter status
```

This verifies your greeter setup and sync status. See the [Configuration guide](./configuration#checking-sync-status) for detailed information.

For optional fingerprint or security-key login, see [Login authentication](./configuration#login-authentication).

### Manual Setup (all distros)

If you prefer to set up the greeter manually, follow these steps:

:::tip

**All users:** You can use `dms-greeter enable` to automate this process instead of following the manual steps below.

:::

1. Edit `/etc/greetd/config.toml` and set `command =` to use dms-greeter:
  ```toml
  [terminal]
  vt = 1

  [default_session]
  user = "greeter"
  command = "dms-greeter --command niri"
  ### Uncomment the below line to run the greeter on Hyprland
  # command = "dms-greeter --command Hyprland"
  ### Uncomment to run the greeter on sway
  # command = "dms-greeter --command sway"
  ### Uncomment to run the greeter on Miracle WM
  # command = "dms-greeter --command miracle-wm"
  ```

2. Disable any existing conflicting greeters:

:::warning

Disabling a greeter while logged in under that greeter will log you out and bring you to a non-graphical TTY.

:::

  ```bash
  sudo systemctl disable gdm lightdm sddm
  ```

3. Enable and start the greeter:
  ```bash
  sudo systemctl enable greetd
  sudo systemctl start greetd
  ```
