### Multi-Line Code Blocks

#### Bash/Shell Scripts

```bash
#!/bin/bash

cd ~/.config/dms
git pull origin main

systemctl --user restart dms
dms-runner reload-config
```

#### Go Configuration

```go
package main

import (
    "github.com/AvengeMedia/dgop/pkg/monitor"
    "log"
)

func main() {
    mon := monitor.New(&monitor.Config{
        Interval:  1000,
        EnableCPU: true,
        EnableGPU: true,
    })

    if err := mon.Start(); err != nil {
        log.Fatal(err)
    }
}
```

#### TOML Configuration

```toml
[compositor]
type = "niri"
auto_start = true

[theme]
mode = "dark"
primary_color = "#805AD5"
accent_color = "#D0BCFF"

[widgets.clock]
enabled = true
format = "24h"
show_date = true

[widgets.system_monitor]
enabled = true
update_interval = 2000
show_cpu = true
show_memory = true
show_gpu = true
```

#### Python Script

```python
#!/usr/bin/env python3

import requests
import json

def get_system_stats():
    response = requests.get('http://localhost:8080/api/v1/stats')
    data = response.json()

    print(f"CPU Usage: {data['cpu']['usage']}%")
    print(f"Memory: {data['memory']['used']} / {data['memory']['total']} MB")
    print(f"GPU Temp: {data['gpu']['temperature']}°C")

if __name__ == "__main__":
    get_system_stats()
```

#### Rust Code

```rust
use serde::{Deserialize, Serialize};
use std::fs;

#[derive(Debug, Serialize, Deserialize)]
struct DmsConfig {
    compositor: CompositorConfig,
    theme: ThemeConfig,
}

#[derive(Debug, Serialize, Deserialize)]
struct CompositorConfig {
    r#type: String,
    auto_start: bool,
}

fn load_config(path: &str) -> Result<DmsConfig, Box<dyn std::error::Error>> {
    let contents = fs::read_to_string(path)?;
    let config: DmsConfig = toml::from_str(&contents)?;
    Ok(config)
}
```

## Collapsible Elements

### Basic Details

<details>
<summary>How do I change the default compositor?</summary>

Edit your configuration file at `~/.config/dms/config.toml`:

```toml
[compositor]
type = "hyprland"
```

Then restart DMS:

```bash
systemctl --user restart dms
```
</details>

<details>
<summary>Where are the log files located?</summary>

DMS logs are stored in:
- System logs: `journalctl --user -u dms`
- File logs: `~/.local/share/dms/logs/`
- Crash reports: `~/.local/share/dms/crashes/`
</details>

<details>
<summary>Can I run DMS on multiple monitors?</summary>

Yes! DMS fully supports multi-monitor setups. Each monitor can have independent widget layouts and configurations.

Configure monitors in `~/.config/dms/monitors.toml`:

```toml
[[monitor]]
name = "DP-1"
primary = true
widgets = ["clock", "system-monitor", "weather"]

[[monitor]]
name = "HDMI-1"
primary = false
widgets = ["music-player", "notifications"]
```
</details>

## Installation Steps

### Step 1: Install Dependencies

Install required system packages:

```bash
sudo pacman -S go gtk4 libadwaita wayland
```

### Step 2: Clone Repository

```bash
git clone https://github.com/AvengeMedia/DankMaterialShell
cd DankMaterialShell
```

### Step 3: Build and Install

```bash
make build
sudo make install
```

### Step 4: Configure Compositor

Edit `~/.config/dms/config.toml` to set your compositor.

### Step 5: Start DMS

```bash
systemctl --user enable --now dms
```

<details>
<summary>Having issues with installation?</summary>

Common problems and solutions:

1. **Missing dependencies**
   - Run: `dms-runner check-deps`
   - Install any missing packages

2. **Compositor won't start**
   - Check logs: `journalctl --user -u dms`
   - Verify Wayland is available: `echo $XDG_SESSION_TYPE`

3. **Widgets not appearing**
   - Validate config: `dms-runner config --validate`
   - Check widget status: `dms-ctl widget list`

Still stuck? Join our [Discord](https://discord.gg/danklinux) for help.
</details>

## API Reference Example

### `dgop.Monitor.GetStats()`

Returns current system statistics.

**Signature:**
```go
func (m *Monitor) GetStats() (*Stats, error)
```

**Returns:**
- `*Stats` - System statistics object
- `error` - Error if collection failed

**Example:**
```go
stats, err := monitor.GetStats()
if err != nil {
    log.Fatal(err)
}

fmt.Printf("CPU: %.1f%\n", stats.CPU.Usage)
```

**Response Format:**
```json
{
  "cpu": {
    "usage": 23.4,
    "cores": 8,
    "frequency": 4200
  },
  "memory": {
    "used": 8589934592,
    "total": 34359738368,
    "percent": 25.0
  }
}
```
