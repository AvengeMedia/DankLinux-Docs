Full, zero-dependency clipboard and clipboard history integration.
- No dependency on external tools, such as cliphist or wl-clipboard.
- Configurable max entry size, retention duration, and automatic cleanup.
- Search API and CLI with pagination, filtering by text, images, or any mime type.
- Automatically ignores data from password managers (such as KeepassXC and 1Password). Preventing sensitive data from being stored in the clipboard history.
- Export & import functionality
- Migration tool for transferring history from cliphist.

### Cursor Control

<figure style={{ textAlign: 'center' }}>
  <img
    src={useBaseUrl('/img/blog/v1.2/cursor-control.png')}
    alt="Cursor Control Settings"
    style={{ maxWidth: '500px', width: '100%' }}
  />
  <figcaption>Customize cursor theme, size, and behavior directly from DMS Settings.</figcaption>
</figure>

Cursor control is available for niri, Hyprland, and MangoWC. This feature provides unified cursor management directly from DMS Settings.

**Features:**
- **Cursor Theme Selection** - Choose from available cursor themes to match your desktop aesthetics
- **Cursor Size Adjustment** - Easily adjust cursor size
- **Compositor-Specific Behavior** - Configure cursor hiding behavior tailored to each compositor:
  - **Niri**: Hide cursor when typing and auto-hide timeout
  - **Hyprland**: Hide on key press, hide on touch, and inactive timeout
  - **MangoWC**: Cursor hide timeout configuration
- **XWayland Compatibility** - Cursor settings are automatically applied to XWayland applications via XResources, ensuring consistent cursor appearance across both native Wayland and xWayland applications

Cursor Control is fully integrated into DMS and supported via dankinstaller, making it easy to set up during initial installation or configuration.

### Clipboard History Migration

1. Cleanup old clipboard history integration:

You can remove the `wl-paste --watch cliphist store` daemon/startup from your compositor configuration.

```kdl
// ~/.config/niri/config.kdl
spawn-at-startup "wl-paste --watch cliphist store" // REMOVE THIS LINE
```

```ini
# ~/.config/hypr/hyprland.conf
exec-once = wl-paste --watch cliphist store # REMOVE THIS LINE
```

2. Migrate clipboard history from `cliphist` to DMS:

Clipboard history will not be automatically migrated from previous versions. You can still access it with the `cliphist` CLI. New clipboard history history located in `~/.cache/DankMaterialShell/clipboard/db` - this is a [bbolt](https://github.com/etcd-io/bbolt) database.

To migrate old history you can follow these steps:

```bash
# First stop DMS
systemctl --user stop dms # Or dms kill , if not using systemd
# Run migration (optionally can override the default path with cliphist-migrate /path/to/db)
dms cl cliphist-migrate
# Start DMS again
systemctl --user start dms # or dms run -d, if not using systemd
```
You can then uninstall cliphist and remove the old database file if desired at `~/.cache/cliphist/db`

`wl-clipboard` can also be removed, although it is useful to keep it for plugins or scripts that may still depend on it.

```bash
# DMS clipboard CLI has feature-parity with wl-clipboard
# To copy text:
echo "Hello, World!" | dms cl copy
# To paste text:
dms cl paste
```
