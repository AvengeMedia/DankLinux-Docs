{/* Wolverine claw tears - the surface looks ripped open, with a molten glow
   showing through three jagged, tapered gashes. Edges are recessed with a dark
   shadow so the dark surface reads as torn away. */}
export const buildGash = (xTop, xBot, halfW, jitter) => {
  const N = jitter.length - 1;
  const pt = (i, side) => {
    const t = i / N;
    const x = xTop + (xBot - xTop) * t;
    const y = 4 + 236 * t;
    const w = halfW * Math.sin(Math.PI * t); // taper to points at both ends
    return `${(x + side * w + jitter[i]).toFixed(1)} ${y.toFixed(1)}`;
  };
  const right = [];
  for (let i = 0; i <= N; i++) right.push(`L ${pt(i, 1)}`);
  const left = [];
  for (let i = N; i >= 0; i--) left.push(`L ${pt(i, -1)}`);
  return `M ${pt(0, 1)} ${right.join(' ')} ${left.join(' ')} Z`;
};

## Frame Mode — Connected & Standalone

> **INSERT LIGHT/DARK IMAGES HERE** — Frame Mode: the bar, dock, popouts, and modals drawn as one connected surface.

Frame Mode is the big one in 1.5. It lets the Dank Bar, dock, popouts, modals, and notifications draw as a single connected surface instead of separate floating pieces.

- **Connected mode** joins shell elements together with SDF-rendered corners and connectors. Popouts and the dock attach to the bar based on which edge it's on.
- **Standalone mode** keeps elements separate, with configurable inset padding and spacing.
- Every surface adapts to whichever side your bar is on, and the dock auto-hides under maximized windows and games.

## DankCalendar Integration

> **INSERT LIGHT/DARK IMAGES HERE** — creating, editing, and navigating events right from the shell.

DMS can now use [DankCalendar](/blog/dankcalendar-release) as a calendar backend over IPC. You can **create, edit, and delete events** directly in the calendar overview — the old khal integration was read-only. There's also a local to-do/task planner, keyboard navigation throughout, an optional week-number display, and a backend selector (Auto, khal, or DankCalendar) in Settings. If `dcal` isn't running, DMS falls back to khal automatically.

## Hyprland Lua Configuration

> **INSERT LIGHT/DARK IMAGES HERE** — Hyprland Lua config parsing, keybinds, and migration.

DMS now supports Hyprland's **Lua** configuration — reading and writing Lua config, Lua-based default keybinds and dispatchers, fractional-scaling display presets, touchpad gestures, and a resize-on-border layout option. Migration tools convert legacy `conf` setups to Lua, and your existing `conf` config keeps working until you migrate.

## MangoWC Support

DMS 1.5 brings full support for MangoWC across the shell, the installer, and the UI tools, with live config reloads, a new workspace pill in the overview, and MangoWC keybinds and dispatchers.

## Material 3 Shadows & Elevation

> **INSERT LIGHT/DARK IMAGES HERE** — the reworked elevation system with global and per-bar shadow controls.

The Material 3 elevation and shadow system has been reworked so shadows stay consistent across notifications, popouts, and other surfaces. A global shadow toggle lives in the Themes tab, with light-color and direction overrides, plus independent shadow settings per Dank Bar.

## Launcher, Spotlight & Clipboard

> **INSERT LIGHT/DARK IMAGES HERE** — the lightweight Spotlight option, plus integrated clipboard and settings search.

There's a new lightweight **Spotlight** launcher alongside the original Dank Launcher. Both get integrated clipboard and settings search, `/d` and `/f` prefixes for directory/file search, launcher history, a toggleable color-picker entry, and `Alt+1/2/3/4` context switching. The **clipboard editor** is back too, with type filters (text/images), search over saved entries, and configurable action buttons.

## Notepad

> **INSERT LIGHT/DARK IMAGES HERE** — the full popout Notepad with side selection and conflict-aware file handling.

The Notepad got a big refactor. A new **popout mode** gives you a full editing surface that docks to the left or right side of the screen, with its own layout, gap, and shortcut settings. There are also stronger safeguards to preserve your work, a banner when a note changes on disk, support for files without an extension, and drag-and-drop tab reordering.

## Network, Battery & Tailscale

> **INSERT LIGHT/DARK IMAGES HERE** — Secret Service integration, connection progress, and the new battery tab.

A lot changed across networking and power this release:

- **Network:** WiFi/VPN secrets retrieved from the D-Bus Secret Service (with unlock support), saved-WiFi state, connection progress indicators, enterprise-WiFi password modals, and stable signal-bucketed ordering.
- **Battery:** A new battery settings tab, percentage and remaining-time toggles, a critical-battery alert system (ported from the DankBatteryAlerts plugin), and Bluetooth device battery detection.
- **Tailscale:** A control-center integration with connect/disconnect, exit-node selection, LAN-access toggles, and live status.

## Greeter & Lock Screen

> **INSERT LIGHT/DARK IMAGES HERE** — the multi-user greeter, auto-login, and FIDO2/U2F security key support.

The greeter got a redesigned multi-user UI with per-user theme sync, an **auto-login** option, and user management from DMS Settings. The lock screen now supports **FIDO2/U2F security keys**, unified PAM services with the greeter, faillock lockout reasons (instead of a generic "incorrect password"), and IME bypass for password entry.

## Headless Installation

The `dankinstall` installer can now run **fully unattended** — no TUI, scriptable for provisioning, dotfile bootstraps, and CI. Passing `--compositor` and `--term` switches it into headless mode:

```bash
dankinstall --compositor niri --term ghostty --yes
```

Choose `niri`, `hyprland`, or `mango` for the compositor and `ghostty`, `kitty`, or `alacritty` for the terminal. Additional flags give you control over the rest of the install: `--include-deps` / `--exclude-deps` to tune optional dependencies (like `dms-greeter`), `--replace-configs` (or `--replace-configs-all`) to deploy named configs, and `--yes` to auto-confirm every prompt. Install logs are written to `$DANKINSTALL_LOG` (falling back to `/var/tmp`) and streamed to stdout.

## Upgrade Notes

:::warning
This release contains changes that may require manual intervention.
:::

### (Breaking) App ID Change

The DMS app ID is now `com.danklinux.dms`. If you have compositor window rules targeting the old app ID, you'll need to update them to the new ID.

### Quickshell 0.3+ Required

DMS 1.5 requires Quickshell 0.3 or newer; support for 0.2 has been dropped. If you install via the Dank Linux repositories or `dankinstall`, this is handled for you — otherwise update Quickshell before upgrading DMS.
